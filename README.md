# 當鋪排班表

這是一個可部署到 Vercel 的動態前端排班工具。資料目前儲存在瀏覽器 localStorage，不需要資料庫。

## 功能

- 獨立新增人員
- 人員可指定 A、B、D 三組
- 人員可依組別查看
- 人員可停用、啟用、刪除
- 人員可在同組內上移、下移調整輪調順序
- 產生目前查看週的週一到週六完整排班
- 已有班表時，重新產生前會先確認
- 所有人模式會重新產生完整班表
- A組、B組、D組模式可只重新產生該組班表，其他組保留不動
- 每次重新產生會重新隨機一次該組順序
- 每人一小時
- 早班、中班、晚班各 4 小時
- 每個時段只會從該組人員中抽選
- 組別內人員不會換組，只會整組輪替到下一個班別
- 切換週時會保持目前的組別查看模式
- 班表固定顯示 09:00～21:00 的完整 12 個時段
- 單組模式仍保留完整班表版面，只在該組本週輪到的時段顯示人員
- 可複製目前畫面排班文字
- 可下載目前畫面 CSV
- 每次開啟網頁會自動清除已結束週次班表
- 網頁開著時，每分鐘檢查一次並清除已結束週次班表

## 班別

- 早班：09:00～13:00
- 中班：13:00～17:00
- 晚班：17:00～21:00

## 組別輪替

基準週：

- A組：早班
- B組：中班
- D組：晚班

下一週自動往下一班輪替：

- A組：中班
- B組：晚班
- D組：早班

再下一週：

- A組：晚班
- B組：早班
- D組：中班

之後每三週循環一次。

## 狀態機

系統使用簡單狀態機控制流程。

- Idle：等待操作
- ManagingPeople：新增、停用、啟用、刪除、調整輪調
- GeneratingSchedule：產生或重新產生班表
- ViewingSchedule：顯示班表
- Error：顯示錯誤訊息

## 部署到 Vercel

1. 到 Vercel 新增專案。
2. 匯入此 GitHub repository。
3. Framework Preset 選 Other。
4. Build Command 可留空。
5. Output Directory 可留空。
6. Deploy。

## 檔案

- `index.html`：頁面結構
- `styles.css`：介面樣式
- `app.js`：狀態機、排班演算法與資料儲存
- `vercel.json`：Vercel 路由設定
- `package.json`：專案資訊
